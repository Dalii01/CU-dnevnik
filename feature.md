# Расширенная аналитика и отчётность

## Контекст

Учителя и администраторы обратились с просьбой улучшить систему аналитики в электронном дневнике. 

Сейчас ученики и родители видят только оценки и посещаемость, но не получают обратную связь о том, как улучшить успеваемость. Учителям нужен инструмент для быстрой оценки успеваемости всего класса, а также удобный способ просмотра оценок и посещаемости по предметам в табличном виде.

Им хочется видеть автоматические рекомендации для учеников, рейтинг класса с возможностью фильтрации и детальные отчёты по предметам. Это позволит быстро оценить ситуацию без лишних кликов.

---

## Требования

### 1. Персонализированные рекомендации в дневнике ученика

**Где:** страница дневника ученика, блок «Рекомендации и аналитика» после статистики

**Для кого:** ученик (свой дневник), родитель (дневник своего ребёнка), учитель и администратор (дневники всех учеников)

**Что отображается:**

Система автоматически анализирует данные ученика и выводит рекомендации четырёх типов:

#### ⚠️ Предупреждение — снижение успеваемости

**Условие:** минимум 4 оценки, средний балл второй половины < первой половины

**Текст:**
```
⚠️ Снижение успеваемости
Ваш средний балл снижается. Первая половина оценок: 4.50, вторая половина: 3.80. 
Рекомендуем уделить больше внимания учёбе.
```

#### 💡 Рекомендация — слабые предметы

**Условие:** есть предметы со средним баллом < 4.0 (минимум 2 оценки по предмету)

**Текст:**
```
💡 Предметы, требующие внимания
Обратите внимание на следующие предметы: Математика (3.5), Физика (3.0). 
Рекомендуем дополнительные занятия.
```

#### 📉 Риск — проблемы с посещаемостью

**Условие 1:** процент отсутствий > 0%

**Текст:**
```
📉 Проблемы с посещаемостью
Процент пропусков: 15.5%. Частые пропуски могут негативно сказаться на успеваемости. 
Старайтесь не пропускать занятия.
```

**Условие 2:** процент отсутствий = 0%

**Текст:**
```
📉 Отличная посещаемость
Отличная посещаемость! Продолжайте в том же духе.
```

#### 🏆 Достижение — успехи в учёбе

**Условие 1:** серия из 2+ пятёрок подряд по одному предмету

**Текст:**
```
🏆 Серия отличных оценок
Отличная работа! У вас серия из 5 пятёрок подряд по предмету Математика. 
Так держать!
```

**Условие 2:** общий средний балл >= 4.5

**Текст:**
```
🏆 Высокая успеваемость
Отличная успеваемость! Ваш средний балл: 4.75. Продолжайте в том же духе!
```

**Важно:** если недостаточно данных для анализа, рекомендации не показываются. Может отображаться несколько рекомендаций одновременно.

### 2. Лидерборд класса

**Где:** страница `/analytics/leaderboard`, кнопка «Рейтинг класса» в шапке сайта

**Для кого:** все роли

#### Для учителя и администратора:

- Просмотр рейтинга всех классов
- Фильтр по классу 
- Фильтр по предмету 
- При выборе предмета — рейтинг только по этому предмету
- При выборе «Все предметы» — общий рейтинг


#### Для ученика и родителя:

- Просмотр рейтинга только своего класса
- Фильтры недоступны
- Автоматическая подсветка строки с текущим учеником

**Таблица содержит:** место, ФИО ученика, класс, средний балл, количество оценок

**Сортировка:** по среднему баллу от большего к меньшему. Ученики без оценок не попадают в рейтинг.

### 3. Отчёты по предметам

**Где:** страница `/reports`, кнопка «Отчёты» в шапке сайта

**Для кого:** учитель, администратор

#### Выбор предмета

Карточка с выпадающим списком всех предметов (название + ФИО учителя) и кнопкой «Показать отчет».

#### Таблица с данными

**Заголовок:** Отчет по предмету: [Название] (Преподаватель: [ФИО])

**Структура:**
- Колонка «Ученик» — ФИО
- Колонка «Класс»
- Для каждой даты — 2 подколонки: «Оценка» и «Посещ.»

**Отображение:**
- Оценки: цветные бейджи (5 — зелёный, 4 — синий, 3 — жёлтый, 2 — оранжевый)
- Посещаемость: ✅ присутствовал, ❌ отсутствовал, `-` нет данных
- Пустые ячейки: `-` серым цветом

**Фильтрация:** учитель видит только своих учеников, администратор — всех.

#### Статистика

Блок из 4 карточек:
1. **Учеников в отчёте** — количество учеников с данными (фиолетовый)
2. **Дней с данными** — количество уникальных дат (синий)
3. **Всего оценок** — общее количество оценок (зелёный)
4. **Средняя посещаемость** — процент с одним знаком после запятой (оранжевый)

#### Пустое состояние

Если по предмету нет данных:
```
⚠️ Нет данных по предмету
По выбранному предмету "[Название]" пока нет данных об оценках или посещаемости
```

## Технические детали

### Сервис аналитики (`AnalyticsService`)

**Основные методы:**

- `get_student_recommendations(student_id)` — возвращает список рекомендаций (type, title, message, icon)
- `get_class_leaderboard(class_name, subject_id)` — возвращает рейтинг с фильтрацией
- `get_student_rank(student_id, class_name)` — возвращает место ученика

**Алгоритмы анализа:**

- `_analyze_grade_trend()` — сравнивает первую и вторую половину оценок
- `_find_weak_subjects()` — ищет предметы со средним < 4.0
- `_find_achievements()` — ищет серии пятёрок и высокий средний балл
- `_analyze_attendance()` — рассчитывает процент пропусков

### Контроллер отчётов (`ReportsController`)

**Метод `_generate_report_data(subject_id, user)`:**

1. Получает учеников (с учётом роли)
2. Собирает оценки и посещаемость по предмету
3. Формирует структуру: `dates` (список дат), `students_data` (данные по ученикам)

### Контроллер аналитики (`AnalyticsController`)

**Маршруты:**
- `/analytics/leaderboard` — общий лидерборд
- `/analytics/leaderboard/<class_name>` — лидерборд класса

**Логика:** определяет роль пользователя, для ученика/родителя автоматически определяет класс, для учителя/админа даёт выбор класса и предмета.
